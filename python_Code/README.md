先说一下跳跃的核心思路（graffiti.py中的jump函数）：

- 首先生成一定数量的踏板（在settings.py中可调），踏板用Group存储

 - 当涂鸦踩到踏板后，开始跳跃，此时更新涂鸦跳跃后的标准高度，使其低于当前屏幕下方的踏板移出Group，不够数目的会生成足够数目

 - 如此往复，刷新屏幕达成跳跃

    

不足之处：

	- 没有以涂鸦为中心，画面不够细腻
	- 这样的判断方法在测试中造成了游戏较为卡顿，需要找到更好的跳跃画面刷新/踏板判定方法
